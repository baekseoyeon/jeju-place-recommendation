# PROJECT

## 프로젝트 소개

`jeju-place-recommendation`은 제주 지역 장소 데이터를 바탕으로 추천 응답을 생성하는 **LLM·RAG 기반 추천 프로젝트**입니다.

이 프로젝트는 인기 장소만 반복 추천되는 구조의 한계를 줄이고, 사용자 질문의 맥락에 맞는 장소와 그 이유를 함께 제시하는 것을 목표로 했습니다. 이를 위해 제주 지역 장소를 **HOT / COOL / COLD** 플레이스로 구분하고, 검색과 프롬프트를 조정해 단순 나열이 아닌 설명형 추천 응답을 만들도록 구성했습니다.

---

## 1. 문제 정의

기존 제주 지역 추천 서비스에는 다음과 같은 한계가 있다고 보았습니다.

- 인기 장소 중심의 반복 추천
- 사용자 질문 맥락이 충분히 반영되지 않는 응답
- 추천 결과는 나오지만 이유 설명이 부족한 구조

이를 보완하기 위해, 단순 인기순 추천이 아니라 **장소 특성과 사용자 질문 조건을 함께 반영하는 추천 구조**를 설계했습니다.

---

## 2. 접근 방식

### 2-1. 추천 기준 재설계

초기에는 기존 추천 방식의 한계를 데이터 흐름 기준으로 확인했습니다.

분석 결과, 사용자 선호는 단순 방문량이나 리뷰 수보다 **연령대, 시간대, 현지인 비율**에 따라 더 뚜렷하게 나뉘는 경향이 있었습니다. 이에 따라 추천 기준을 다시 정리해, 이용 연령층·시간대·현지인 비율을 함께 고려하는 방향으로 재구성했습니다.

### 2-2. 장소 구분 기준

추천 결과가 한쪽으로 치우치지 않도록 제주 지역 장소를 아래와 같이 구분했습니다.

- **HOT Place**: 이미 많이 알려진 인기 장소
- **COOL Place**: 덜 알려졌지만 추천 가치가 있는 장소
- **COLD Place**: 현재 추천 우선순위가 낮은 장소

이 기준은 단순 라벨링이 아니라, 질문 맥락에 따라 어떤 장소를 우선 제시할지 판단하는 기준으로 활용했습니다.

### 2-3. 사용자 질문과 데이터 구조 차이 보정

프로젝트를 진행하며 확인한 핵심 문제 중 하나는 **사용자 표현 방식과 실제 데이터 구조가 다르다**는 점이었습니다.

예를 들어 사용자는 `용두암 근처 흑돼지`처럼 질문하지만, 실제 데이터는 `용담이동`, `육류/고기`처럼 행정구역·업종 분류 중심으로 저장되어 있었습니다.

이를 보완하기 위해 다음 작업을 진행했습니다.

- 사용자 질문 표현과 데이터 분류 체계 차이 확인
- Kiwi 사용자 사전을 활용한 동의어 정리
- 질문 의도에 맞게 분류 체계 보완
- 검색 단계에서 실제 질문 표현이 반영되도록 텍스트 구조 조정

---

## 3. 구현

### 3-1. 데이터 전처리

공모전 제공 원본 데이터는 그대로 사용하지 않고, 추천 서비스에 맞게 다시 정리했습니다.

주요 전처리 작업은 다음과 같습니다.

- 장소 관련 텍스트 정리
- 카테고리 및 속성 정보 재구성
- 추천에 필요한 컬럼 정리
- HOT / COOL / COLD 기준 반영
- 검색 가능한 형태로 데이터 구조화
- 문장 또는 chunk 단위로 재구성

**관련 코드**

- `run/sentences.py`: 문장 단위 데이터 정리
- `run/chunking.py`: chunk 단위 전처리 및 재구성

### 3-2. RAG 파이프라인

추천 구조는 **검색 + 생성** 방식으로 구성했습니다.

전체 흐름은 다음과 같습니다.

1. 사용자 질문 입력
2. 관련 장소 정보 검색
3. 검색 결과 정리
4. LLM이 추천 결과와 이유를 포함한 응답 생성
5. 웹 인터페이스에서 출력

검색 구성은 아래와 같습니다.

- **BM25** 기반 검색으로 핵심 키워드 반영
- **임베딩 기반 검색**으로 의미적으로 유사한 장소 탐색
- **FAISS 인덱스**를 활용한 벡터 검색 구성

즉, 이 프로젝트는 **키워드 검색 + 의미 유사도 검색 + LLM 생성**을 결합한 형태입니다.

**관련 코드**

- `run/embeddings.py`: 임베딩 생성 및 저장
- `run/chunking.py`: 검색 가능한 텍스트 단위 구성
- `run/sentences.py`: 원본 텍스트 정리

### 3-3. Prompt Engineering

프롬프트는 검색 결과를 단순 나열하지 않고, **추천 결과 + 추천 이유 + 질문 조건 반영**이 함께 나타나도록 조정했습니다.

중점적으로 본 부분은 다음과 같습니다.

- 지역, 음식 종류, 분위기 등 질문 조건 반영
- 검색 결과 기반 추천 생성
- 추천 이유가 포함되도록 응답 형식 조정
- HOT / COOL / COLD 구분 반영
- 사용자 표현과 데이터 표현 차이 보완

프롬프트는 한 번에 확정한 것이 아니라, 테스트 케이스를 기준으로 반복 수정하며 응답 방식을 보정했습니다.

### 3-4. 웹 프로토타입

검색과 생성 결과를 확인할 수 있도록 **Streamlit 기반 웹 프로토타입**을 구현했습니다.

사용자는 자연어로 질문을 입력하고, 추천 결과와 추천 이유를 함께 확인할 수 있도록 구성했습니다.

**관련 코드**

- `run/alphastorm_app.py`: Streamlit 기반 웹 애플리케이션 실행 파일

### 3-5. 코드 구성

현재 실행 관련 코드는 `run/` 폴더에 정리되어 있습니다.

- `run/embeddings.py`: 장소 데이터 임베딩 생성, 벡터 저장 및 검색 준비
- `run/chunking.py`: 데이터를 chunk 단위로 분할 및 정리, 검색 가능한 텍스트 단위 구성
- `run/sentences.py`: 문장 단위 텍스트 정리, 장소 설명 텍스트 전처리
- `run/alphastorm_app.py`: Streamlit 기반 웹 실행, 사용자 질문 입력 및 추천 결과 출력
